It is called the <b>status register</b> or <b>condition</b> <b>code</b> <b>register</b>. It is an 8 bit register which has its bits set independently according to particular result of a test done on the accumulator.</p><p>One such test is the <b>overflow</b> and you can identify this register in figure 8.1 of module 8. If after an instruction as been executed in the accumulator an overflow occurs, then a special bit in the status register called the <span class="spCh spChx2018">&#x2018;</span>overflow bit<span class="spCh spChx2019">&#x2019;</span> or <span class="spCh spChx2018">&#x2018;</span>flag<span class="spCh spChx2019">&#x2019;</span> will be set to a logic 1 accordingly.</p><p>The purpose of remembering the status of a previous operation is to allow <b>branching</b> to another part of a program to occur. For example if the overflow flag = 1, <b>branch</b> to another part of the program. If not, carry on normally. An instruction satisfying this criteria is<b> branch if overflow set (BVS).</b></p><h1><a id="id11" name="id11"><!--id11--></a>9.4 Carry conditions</h1><p>Another bit in the status register is set to a logic 1 if there is a <span class="spCh spChx2018">&#x2018;</span>carry<span class="spCh spChx2019">&#x2019;</span> from a previous operation in the accumulator.</p><h2><a id="id12" name="id12"><!--id12--></a>9.4.1 Carry examples</h2><p>Consider the addition of two correct size negative numbers:</p><p class="indent"><a name="graphics10"/><img alt="graphics10" class="fr1" height="98" src="module09_files/152f97d1_267x98.jpg" style="border:0px; vertical-align: top" width="267"/></p><p>Since the answer must be stored in an 8 bit register, the carry bit can be stored in the status register.
