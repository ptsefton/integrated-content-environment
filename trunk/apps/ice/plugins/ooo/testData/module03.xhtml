Module overview

In this module you will learn how to use the Integrated Content Environment (ICE) system to develop your content.

3.1 Getting started

Software Development Services (SDS), in the Distance and e-Learning Centre (DeC), will set up your course directory structure in the repository with the relevant access permission. They will use your alias and create a Subversion password. It is recommended that you visit SDS and change the allocated password. (It is intended that the ICE system will use your USQConnect username and password with future development.) Inform SDS if you require access to other courses.

A sample course will be included, containing a few documents to base your content structure on.

3.1.1 The ICE server and OpenOffice.org (OOo) Writer

The ICE server and OpenOffice.org Writer must be started to gain access to the system.

Your installation of OOo Writer may have included the load on startup option. If that is the case, you will see a Quickstarter icon on the Windows Taskbar which means it is running.

If the OOo Writer Quickstarter is not available, start Writer from Start...Programs.

For Windows installations, run the server by clicking on the shortcut to Ice_Server.exe on your Windows Taskbar.

3.1.2 Login or work offline

To access the repository you need to login using your USQConnect username and password.  It is also possible to access ICE content using the wireless VPN (Virtual Private Network) from home if it is available to you. The Login screen also provides an option to work offline. This means you can still use the ICE system to view and update content, however you can’t synchronize with the repository.

Login

  1. Open an Internet browser.

  2. First time user or unsure of full URL: Insert the following URL in the Address text box: http://localhost:8000/.

    Existing user: If you have bookmarked or remember the full URL to your content, include it in the Address text box.

    A Login screen displays.

  3. Insert your username and password for the SVN repository then click the Submit button.

  4. First time user: Follow the links to your content.

    Bookmarked URL: Your content will display.

Work offline

  1. Open an Internet browser.

  2. First time user or unsure of full URL: Insert the following URL in the Address text box: http://localhost:8000/.

    Existing user: If you have bookmarked or remember the full URL to your content, include it in the Address text box.

    A Login screen displays.

  3. Click the Work Offline button.

  4. First time user: Follow the links to your content.

    Bookmarked URL: Your content will display.

3.1.3 System timeout

The ICE system will timeout after approximately 15 minutes of inactivity. You will have to login to resume access.

3.2 Understand the environment

3.2.1 Toolbar functions

There is a toolbar at the top of the ICE website which contains buttons to call ICE functions. The toolbar is not exported with the content.

Button

Description

Re-render

Loads all modified and new files into the ICE site. After you modify a Writer document, save the file. Click the Re-render button to display the updated content.

Note: While browsing the content, ICE will automatically re-render a document about to be viewed if it has changed.

Force re-render

Re-loads all files into the ICE site. This function is rarely required.

Sync

Used to Synchronize with the server and other users. It checks for new and changed files and sends all modified files to the server.

Get changes

Downloads any new or changed files on the server. It is a good idea to use this function before making changes to your local content to ensure you are working on the latest version.

Revert all

Revert all local changes made to the content since last using Sync.

Revert

Revert local changes to the file being viewed since last using Sync.

Check links

Checks links in the content. Used to validate links to content within the content structure and also displays any external links used. It is impossible for ICE to verify external links as firewalls and server problems can occur.

Organize

Organize the manifest content structure, insert the content title, change/set content titles, hide files, delete files and set the home page.

Note: The Delete option does not only remove reference to the file in the manifest, but deletes it from the content structure.

Export

Creates a standalone HTML version of the content which is zipped. The zip file is saved in the root of the content structure. Click the Submit button after changes to the organization.

Template

The options available from the dropdown list are Default, CD or Web.

Note about using Sync and Get changes

Before using the Sync or Get changes functions, close all open documents to avoid possible problems with open files being locked.

3.2.2 The manifest

A file called manifest.xml is automatically created in the root of the content structure when ICE is run with content for the first time.

The manifest is a text file that contains the structure information for the site. You will not need to modify this file.

3.2.3 The skin

The skin is a wrapper to display the content. It is possible to include a different skin for content by editing the files in the package .skin folder

3.3 File handling operations

ICE uses the Subversion version control which provides additional file handling features to normal operating system commands to manage content.

“Subversion can access its repository across networks, which allows it to be used by people on different computers. At some level, the ability for various people to modify and manage the same set of data from their respective locations fosters collaboration. Progress can occur more quickly without a single conduit through which all modifications must occur. And because the work is versioned, you need not fear that quality is the trade-off for losing that conduit—if some incorrect change is made to the data, just undo that change.”

(Source: http://svnbook.red-bean.com/en/1.1/svn-book.html#svn-ch-1-sect-1)

3.3.1 Access to another course

If you are granted access to another course in the repository, it is necessary to update your directories to be able to view the content.

Ensure that the ICE server and OpenOffice.org Writer are started.

  1. Open an Internet browser and insert the following URL in the Address text box: http://localhost:8000/.

  2. Login using your Subversion username and password.

  3. Click the ‘packages’ directory link.

  4. Click the ‘Get changes’ button in the toolbar.

  5. Be patient while ICE downloads new files and folders from the server.

3.3.2 Delete a file from your directory structure

Ensure that the ICE server and OpenOffice.org Writer are started.

  1. Open an Internet browser and insert the following URL in the Address text box: http://localhost:8000/.

  2. Login using your Subversion username and password.

  3. Browse to your content.

  4. From the ICE toolbar, click the Organize button.

  5. Select the Delete check box for the file or files you wish to delete.

  6. Click the Submit button.

  7. From the ICE toolbar, click the Sync button to commit the deletion to the repository.

    If you have made a mistake, click the Revert button on the ICE toolbar to undo the deletion.
Note

Deleting a file via Organize in ICE will physically remove the file from your system.

Do not delete a file in any other way (e.g. keyboard key or shortcut). If you do, the file will reappear the next time you Sync or use Get changes in the ICE site.

3.3.3 Conflicts

A conflict occurs when the same file you are editing has changed in the repository since you last used Sync or Get changes. ICE deals with this by making a copy of your modified file and replaces the modified file with the latest copy from the repository.
Example  when a conflict occurs

If a conflict occurs on module01.odt your modified file will be renamed to ‘myChanges_module01.odt’. The ‘module01.odt’ file will be updated with the latest file in the repository. A message in the browser will advise when this occurs.
