Module overview

In this module you will learn how to use the Integrated Content Environment (ICE) system to develop your content.

1 Start the ICE server and OpenOffice.org (OOo) Writer

The ICE server and OpenOffice.org Writer must be started to gain access to the ICE system in your browser.

Windows users

If OOo is automatically loaded at start-up, you will see a Quickstarter icon on the Windows Taskbar which means it is running. Check if the start-up option has been set using the following process.

Tools...Options...OpenOffice.org (expand if necessary)...Memory. From the right-hand pane, select the option to Load OpenOffice.or during system start-up.

If the Quickstarter is not available, start Writer from the Start menu on the Windows Taskbar (e.g. Start...Programs...OpenOffice.org 2.0...OpenOffice.org Writer).

Start the ICE server from the Start menu on the Windows Taskbar (e.g. Start...Programs...ICE...ICE server) or from a shortcut if you have one created on your Windows Taskbar or Desktop.

Mac users

To start OpenOffice.org, run the application file via Finder (e.g. Finder...Applications...OpenOffice.org2.0.app).

To start the ICE server, run the application file via Finder (e.g. Finder...Applications...Ice.app).

2 Login or work offline

To access the repository you need to login using your username and password. The Login screen provides an option to work offline. This means you can still use the ICE system to view and edit content, however you can’t synchronize with the repository.
Note

The ICE system will timeout after approximately 15 minutes of inactivity. You will have to login to resume full access.

2.1 Login

  1. Start your ICE Server.

  2. Open an Internet browser.

  3. First time user or unsure of full URL: Insert the following URL in the Address text box: http://localhost:8000/

    Existing user: If you have bookmarked or remember the full URL to your content, include it in the Address text box.

  4. When the Login screen displays, insert your username and password then click the Submit button.

  5. First time user: Follow the links to your content.

    Bookmarked URL: Your content will display.

2.2 Work offline

  1. Start your ICE Server.

  2. Open an Internet browser.

  3. First time user or unsure of full URL: Insert the following URL in the Address text box: http://localhost:8000/.

    Existing user: If you have bookmarked or remember the full URL to your content, include it in the Address text box.

  4. When the Login screen displays, click the Work Offline button.

  5. First time user: Follow the links to your content.

    Bookmarked URL: Your content will display.

3 The manifest and skin

A file called manifest.xml is automatically created in the root of the content structure when ICE is run with content for the first time.

The manifest is a text file that contains the structure information for the site. Use the package organizer to setup the structure. You will not modify the manifest file directly.

The skin is a wrapper to display the content. For example, the logo, layout and CSS (Cascading Style Sheet). The skin is customizable.

4 Navigation

Various navigation options are available within the ICE website.

Left-hand menu – The default display is first level items. When an item is accessed, second level items will display, if applicable, indented as shown in the example.

Breadcrumb trail – The breadcrumb trail displays above the content on a screen. Use to see where you are in the content structure.

Contents – Access from the Contents link at the top of the left-hand menu. The first and second level items for the whole content are displayed.

Mini table of contents – Displays first and second level items below the title of each screen.

5 Toolbar functions

A toolbar at the top of the ICE website contains buttons to call ICE functions. The toolbar does not stay with the content when exported.

5.1 Re-render

Load all modified and new files into the ICE site. After you modify a document, save the file. Click the Re-render button to display the updated content.

While browsing the content, ICE will automatically re-render a document about to be viewed if it recognizes that changes have occurred.

5.2 Get changes

Close your documents before using this function. Download any new or changed files from the server. It is a good idea to use this function before making changes to your local content to ensure you are working on the latest version.

5.3 Sync

Close your documents before using this function. Synchronize with the server and other users. It checks for new and changed files and sends all modified files to the server. Remember to perform a Syn at the end of each editing session.

5.4 Organizer

The Package organizer is used to arrange the content structure for the manifest and edit properties.

Links toolbar functions

Expand all and Minimize all – Expand or minimize the display of children of all parent items.

Edit properties – Edit the upper level properties of the package.

  • Title – If required, change the package title displaying in the website. Click the Edit properties link...Edit the title...Click the OK button.

  • Set as Home page – Select this option for the list of contents to display as the opening screen of the website. Click the Edit properties link...Select or deselect the Set as Home page option...Click the OK button.

Icons

Move the selected item/s before the following item.

Move the selected item/s to the next level (child) in relation to the parent item.

 

When there are children items, a plus sign icon will appear. Use this icon to expand the items.

 

While the items are expanded, a minus sign icon will appear. Use this to minimize the items.

Other functions

View – View the item content in the ICE website.

Edit properties – Access the item properties. The options will vary depending on the type of file.

  • Title – The title included here is from the document property field. It is displayed in the website navigation. You may wish to shorten the title in the web version for the navigation. If you change the title properties in the document, you also need to edit the properties here in the organizer. Click the Edit properties link...Edit the title (if document properties have been changed, it will display here to await acceptance)...Click the OK button.

  • Hide – Hide the item from the website navigation. Click the Edit properties link...Select or deselect the Hide option...Click the OK button.

  • Set as Home page – Select for the item to display as the opening screen of the website. Click the Edit properties link...Select or deselect the Set as Home page option...Click the OK button.

  • Render (ICE document) – This option is available for Word documents. If the option is selected, the document will be converted in the website. If the option is deselected, the document will display as a Word document. Click the Edit properties link...Select or deselect the Render (ICE document) option...Click the OK button.

5.5 ICE file manager

Manage your ICE content files using this function instead of using your Windows file managers.

Links toolbar functions

Update selected – Update the selected item with the most recent version from the server. Select the relevant checkbox...Click the Update selected link.

Sync selected – Send the selected new or modified item to the server. Select the relevant checkbox...Click the Sync selected link.

Re-render selected – Re-render selected items only. Select the relevant checkbox...Click the Re-render selected link.

Force re-render selected – Force a re-render of selected items whether they have been changed or not. For example, use a force re-render to implement development fixes in ICE.

Add new folder – Creates a new folder to add content. Use the file manager to navigate to the required location then click the Add new folder link. Under the links toolbar, a New folder name option will display. Type the required folder name then click the Add button. The new folder will be added to the displayed list. Click the folder name to open it.

Cut selected – Use when an item needs to be moved from one location to another. Select the relevant checkbox...Click the Cut selected link...Navigate to the required location...Click the Paste link. The item will not be removed from the original location until it has been pasted in the new location.

Copy selected – Copy an item so it can be added to another location. Select the relevant checkbox...Click the Copy selected link...Navigate to the required location...Click the Paste link.

Paste – This function will only be available after using Cut selected or Copy selected. Navigate to the required location then click the Paste link.

Delete selected – Permanently remove an item. Select the relevant checkbox...Click the Delete selected link.

Revert selected – Revert an item to the previous repository version. All local changes that have not been synchronized with the repository will be lost. Select the relevant checkbox...Click the Revert select link.

Advanced options – Use to repair Writer documents. For example, apply the latest ICE Writer template to documents based on a previous version. After the repair, check formatting and apply styles as required. For example, a heading that was previously h1-chapter requires the h1n style applied.

  • Use default template file – This template is for documents that do not require outline numbering based on a module number. Click the Use default template file link...Ensure the required file path is displayed on the Repair selected link...Select the file/s...Click the Repair selected link.

  • Set selected file as template file – Select a specific template file. Using the file manager, navigate to the template file and select it...Click the Set selected file as template file link...Ensure the required file path is displayed on the Repair selected link...Navigate to the file/s to be repaired...Select the file/s...Click the Repair selected link.

  • Repair selected – The suffix (without template file) indicates a template for repairing the document has not been selected. When a template file has been selected, the suffix will change to include the path of the file.

Create OpenOffice.org Writer file or Create Word file – Creates a new Writer or Word file from an ICE template. Use the file manager to navigate to the required location then click the Create OpenOffice.org Writer file or Create Word file link. Under the links toolbar, the options for inserting a filename and selecting a template type will display. Type the required filename...Select the template type from the Type dropdown list...Click the Create button. The new file will be added to the displayed list. Click the filename to open the document and start working.

  • Template type

    Default – For documents that do not require outline numbering based on a module number.

    Module – For documents that require outline numbering based on a module number.

Upload file – Upload one file at a time into your ICE content structure. Use the file manager to navigate to the required location then click the Upload file link. Under the links toolbar, an Upload file from your file system option will display. Click the Browse... button. Locate the file then click the Open button. Click the Upload button.

Icons

Folder – Click the link next to the icon to take you to the contents of the folder.

File – Open the file in the ICE website.

Parent – Navigate up one level in the structure.

All checkboxes – Select/deselect all of the items displayed with one click.

Checkbox – Select/deselect individual items displayed.

Other functions

File name link – Word and Writer filenames include a link so you can open the file directly from the file manager in its native program.

Log comment – It is optional to include a comment when synchronizing a selected item with the repository. Select the relevant checkbox...Type a comment in the Log comment text box...Click the Sync selected link. During collaboration between two or more authors, a comment could assist with explaining what has been edited in a document. This function is not available when using the Sync toolbar button to synchronize the entire site.

Path – The file manager path of the displayed files and folders. The path links can quickly take you to higher level directories.

Refresh – Refresh the view of the file manager screen. For example, after looking at a log which displays under an item, click the Refresh link to remove the log list.

View – Exits the file manager to take you to the folder of the displayed path.

Name – Displays the name of each item in the selected directory.

Date – Represents the date you retrieved the item or changes from the repository or when you last modified the item. Format is year-month-day.

Commands

  • Rename – Change the name of an item. Click the Rename link for the item name to become editable and the Rename link changes to a Save button. Change the item name then click the Save button.

  • Log – View the last three revision history items. The log will display below the item. Use the Refresh link to close the log.

  • Revert and Sync – After deleting a selected file, you are given the options to revert and sync. Revert will revert the item to the previous repository version. Sync will synchronize the deletion with the repository.

Status

  • Added – The item is marked as added for the repository.

  • Modified – The item has been modified compared to the same item in the repository.

  • PropModified – The Subversion properties of the item (e.g. HTML or PDF) is different to the repository version.

  • Out-of-date – The item is older than the version in the repository.

  • Conflict – The item is modified and out-of-date compared to the repository version.

Action – Displays a summary of the action performed.

Details – Displays further details about the action performed.

5.6 Check links

Validate links between content in your content structure. Invalid internal links will be listed so you can correct them.

A list of external links used will also be displayed. It is impossible for ICE to verify external links, as firewalls and server problems can occur. You will need to manually check these yourself to ensure they are valid.

5.8 Export

After a template has been selected, the export function creates a standalone HTML version of the content being viewed. The exported content is in a Zip file which is saved in the root of the content structure. For further details, see 2.10 Content export from ICE.

5.9 Logout and login

When you have finished working on your content, use this button to logout of the system. A timeout will occur if the system has been inactive for approximately 15 minutes. The Logout button will be replaced with a Login button. You will still be able to work on your content and use the system functions except for Get changes and Sync. Login to regain access to Get changes and Sync.

5.10 Content status

The icons at the right-hand end of the toolbar will change depending on the status of your content.

Conflict – red with white exclamation mark

Local changes – blue with white tick

Offline – gray with white emblem

Offline modifications – blue with white emblem

In sync – green with white tick

Out of sync – yellow with white question mark

6 Conflicts

A conflict occurs when the same file you are editing has changed in the repository since you last used Sync or Get changes. ICE deals with this by making a copy of your modified file and replaces the modified file with the latest copy from the repository.
Example – when a conflict occurs

If a conflict occurs on module01.odt your modified file will be renamed to myChanges_module01.odt. The module01.odt file will be updated with the latest file in the repository. A message in the browser will advise when this occurs.

7 Printable version of content

A PDF icon will display on the top right-hand corner of content that has a printable version. When you click the icon a PDF will open for printing.

If a PDF icon is on the Contents screen, your content structure contains a master document. A master document can contain the combined content for printing if it has been set up.

8 Access new repository content

If you are granted access to more content in the repository, it is necessary to update your directories to be able to view the content.

  1. Ensure that the ICE server and OpenOffice.org Writer are started.

  2. Open an Internet browser and insert the following URL in the Address text box: http://localhost:8000/.

  3. Login using your username and password.

  4. Click the packages directory link.

  5. From the Packages screen, click the Get changes button on the toolbar.

  6. Be patient while ICE downloads new files and folders from the server.

10 Content export from ICE

Use the following process to export ICE content.

  1. Log in to the ICE website for your content (see 2.2.1 Login) to finish editing and/or proofing your content.

  2. Click the Check links button on the ICE toolbar. If you have invalid links, correct them before proceeding. Ensure external links are still valid.

  3. From the ICE toolbar, select a template from the Template dropdown list.

  4. Click the Export button. This process packs up all of the converted content into a Zip file. The Zip file can be located at the root level of your content. For example:

    FET8660_2006S1_Default.zip
    FET8660_2006S1_CD.zip
    FET8660_2006S1_Web.zip

11 ICE content for a LMS

The Zip file containing exported ICE content can be uploaded into a LMS (Learning Management System).

11.1 Upload content into WebCT

  1. Follow the export process in 2.10 Content export from ICE.

  2. Open a browser and insert the URL for your WebCT Institution.

  3. Click the Log In button. The Log In screen will display.

  4. Enter your user name and password then click the OK button. Your Course List will display.

    If a Warning-Security screen displays, select Yes if you want the screen to display each time of entry or select Always so the message does not appear again.

  5. Click the required course link. An information page about the tabs may display. If it does display, either click the Start button or select the option to not show the page again then click the Start button to proceed.

  6. Click the Build tab then click the Basic View tab.

  7. From the right-hand side of the same row where the Basic View tab is located, click the File Manager link. The File Manager displays.

    Click the Upload File button. The Upload File screen displays.

    In the File section, click the Browse button, locate the path, select the Zip file then click the Open button.

    Click the checkbox to select Automatically expand uploaded Zip files.

    From the dropdown list in the Character Set section, select Unicode (UTF-8).

    Click the Save button. Be patient while this process occurs. When the upload is complete, you will be taken back to the File Manager. You will notice that the uploaded Zip file will be listed.

  8. Click the Basic View tab.

  9. From the Add to Home Page section on the left-hand side, click the Content File link.

  10. Click the Select File button. The File Browser will display.

    Locate the Zip file you uploaded and its related folder. The folder will be sitting directly above the Zip file. Click the folder name. The contents will display.

    Click the checkbox next to the opening file (e.g. default.htm) then click the Add Selected button. The content item will be added to the Organizer Page.

  11. Click the content name link then select Edit. The Edit Content File screen displays.

    In the Title textbox, replace the title with Course materials then click the Save button. You will be returned to the Organizer Page.

  12. To reposition the content name link, click the name then use the Move arrows. Repeat until it is in the correct position.

  13. Click the Student View tab and test the content link.

11.2 Replace uploaded content in WebCT

Currently the Replace function in WebCT is not working correctly. Previously uploaded content needs to be removed before uploading the replacement content.

  1. Perform Steps 1–6 in 2.11.1 Upload content into WebCT.

  2. To remove the content name link from the Organizer Page, click the arrow next to it and click the Remove link. This only removes the link to the content but does not remove the content from the File Manager.

  3. From the right-hand side of the same row where the Basic View tab is located, click the File Manager link. The File Manager displays.

    From the File Manager, click the checkboxes next to the Zip file and its related folder, click the Delete button then click the OK button.

    Click the Upload File button. The Upload File screen displays.

    In the File section, click the Browse button, locate the path, select the Zip file then click the Open button.

    Click the checkbox to select Automatically expand uploaded Zip files.

    From the dropdown list in the Character Set section, select Unicode (UTF-8).

    Click the Save button. Be patient while this process occurs. When the upload is complete, you will be taken back to the File Manager. You will notice that the uploaded Zip file will be listed.

  4. Perform Steps 8–13 in 2.11.1 Upload content into WebCT.
